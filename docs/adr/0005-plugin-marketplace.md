# Add a Claude Code plugin marketplace as a second, secondary install path

This repo's primary distribution is `npx skills@latest add gio-del/skills`, which works across any Agent-Skills-compliant tool and lets someone cherry-pick individual skills. We added `.claude-plugin/marketplace.json` alongside the existing `.claude-plugin/plugin.json` so Claude Code users can also install via `/plugin marketplace add` — but deliberately kept it a single plugin entry (`source: "./"`, inheriting the existing `plugin.json`'s skill list) rather than splitting into one plugin per category or per skill. Per-skill selection is already solved by the `npx` picker; re-solving it as N marketplace entries would mean two selection UIs that can drift out of sync, and would add a step to every future "add a new skill" change. `version` lives only in `plugin.json` (bumped per `CONTRIBUTING.md` whenever a skill is added or changed) — `marketplace.json` has no `version` of its own, so it can't fall out of sync with it. `npx` stays the documented primary path in the README; the marketplace is presented as an alternative, since it's Claude-Code-specific where `npx` is agent-agnostic. This is hard to reverse once people depend on the `/plugin install gio-del-skills@gio-del-skills` command, and non-obvious — a future contributor could reasonably assume finer-grained marketplace entries were intended.
