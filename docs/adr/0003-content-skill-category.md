# Introduce a `content` category for skills that produce audience-facing output

`/design-system` and `/slides` don't fit either existing category: `engineering` is for skills that build and change a target *codebase*, `productivity` is for planning and non-code work for the user's own benefit, but these skills generate polished artifacts (a reveal.js theme, a slide deck) meant for an external audience. We introduced `skills/content/` for this and future audience-facing-output skills rather than force-fitting into `engineering` (mechanically similar — file generation — but conceptually different intent) or stretching `productivity` past its definition. Same justification as ADR 0002's `meta` category: hard to reverse once more content skills accumulate, and non-obvious — a future contributor could assume only `engineering`/`productivity`/`meta` exist.
